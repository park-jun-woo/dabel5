## 「自己複製だと言いながらチップはどこから買うのか？」

これまでの記事でミラー、構造材、[タービン](/ja/why/why-turbines/)、[バッテリー](/ja/why/why-iron-nickel-battery/)、[熱管理](/ja/why/heat-flow/)まで——小惑星Fe-Niで作れることを示した。自己複製ループがほぼ閉じた。

ほぼ。

**AIチップはまだ地球から輸入している。** ダイソンモジュールの自律運用——採掘ロボット制御、軌道調整、精錬工程管理、居住区生命維持——はすべてAIが行う。チップがなければモジュールは目の見えない巨人だ。

「小惑星にリチウムがない」がリチウムイオン電池のゲームオーバーだったように、**「宇宙でEUVを作れない」が最先端3nmのゲームオーバーだ。**

ではどの工程でチップを作るのか？

---

## なぜ最先端3nmではないのか

半導体工程の核心はリソグラフィ——光で回路パターンをウェハーに刻む工程だ。

| 項目 | 28nm | 3~5nm（最先端） |
|------|------|---------------|
| リソグラフィ | **ArFイマージョン**（ニコン、キヤノン、ASML） | **EUV**（ASML独占、1台数千億円） |
| 装置の入手性 | 成熟市場、中古豊富 | 極めて限定的、輸出規制対象 |
| 設計複雑度 | シングルパターニング | マルチパターニング（極めて複雑） |
| ファブ建設費 | ~$3~5B | ~$20~30B |
| 歩留まり | 高い（10年+の実績） | 初期は低い |

EUV（極端紫外線）スキャナーは**地球全体でASML一社だけが作っている。** オランダ・アイントホーフェンの単一工場。輸出規制対象。米国・日本・オランダの同盟が中国への販売を阻止しているあの装置だ。これを宇宙で再現する？不可能だ。

EUVが不要な最も強力な工程。それが**28nm**だ。

「7nmはArFでできるのでは？」——できる。マルチパターニングという技法でArF光を複数回照射してより微細なパターンを作る。だが設計複雑度が爆発的に増大し歩留まりが急落する。宇宙で歩留まり管理のための人員とインフラを備える前には非現実的だ。

「65nmならもっと簡単に作れるのでは？」——その通り。だがチップあたりの性能が低すぎる。同じ仕事をさせるには物量が激増し、物量が増えれば配線・パッケージング・冷却が比例して複雑になる。簡単に作れてもシステム全体が難しくなる。

**28nm = EUVなしで達成できる最適な集積度。**

## これは理論ではない——Google TPU v1

「28nmでAIを動かせるのか？」

Googleが2015年に答えを出した。**[TPU v1](https://arxiv.org/abs/1704.04760)。** 28nm工程で製造し、自社データセンターに10万個以上配備した実戦AIアクセラレータ。

| 項目 | Google TPU v1（実測） |
|------|---------------------|
| 工程 | 28nm |
| 構造 | 256 × 256 シストリックアレイ |
| 演算 | 92 TOPS (INT8) ≈ 23 TFLOPS (FP16) |
| 消費電力 | 実運用 ~75W |
| シリコン活用率 | **90%+** |

シストリックアレイ（systolic array）という構造が鍵だ。GPUは汎用チップなのでシリコンの70%が制御ロジック、キャッシュ、スケジューラーに使われる。実際に行列演算を行うのは30%。シストリックアレイは行列乗算のみを行うように設計された構造で、**シリコンの90%以上が実際の演算に使われる。**

AIだけを動かすならGPUの汎用オーバーヘッドは全部無駄。TPUはその無駄をなくしたチップだ。

そしてこれは論文上の提案ではない。**AlphaGoを動かしたチップだ。** Googleデータセンターで数年間実サービスに投入されたハードウェア。

## 「4.6倍電力を食う？」

現在の地球最高性能AIチップ、NVIDIA H100。4nm工程、990 TFLOPS、消費電力700W。

TPU v1一枚は23 TFLOPS。H100一枚と同じ演算をするには？

```
990 TFLOPS ÷ 23 TFLOPS = 43枚

43枚 × 75W = 3,225W ≈ 3.2 kW
```

| | TPU v1 × 43枚 | H100 × 1枚 |
|---|---|---|
| FP16合計 | ~990 TFLOPS | ~990 TFLOPS |
| 総電力 | **3.2 kW** | 700W |
| 電力比率 | **4.6倍** | 基準 |

4.6倍。地球では致命的な格差だ。電気代がデータセンター運営費の30~40%を占める世界で4.6倍の電力差は即破産。

**宇宙では？**

ダイソンモジュール1基 = 370 MW。3.2 kWは370 MWの**0.00086%。** ミラー面積にして2.4 m² — ダイソンミラー1 km²の画素一つ分。

地球では電力がお金だ。**宇宙で電力はミラー面積だ。** ミラーは小惑星Fe-Niを伸ばして作る。

[前の記事でタービンが太陽光パネルに勝った](/ja/why/why-turbines/)のと同じ論理構造だ。地球基準では劣る選択が宇宙基準では唯一の選択に逆転する。**基準が違えば答えが変わる。**

## モジュール1基 = H100 3万枚級データセンター

モジュールの370 MWのうち30%をAI演算に配分すると：

```
111 MW ÷ 75W/チップ = ~1,480,000枚（148万枚 TPU v1）

148万枚 ÷ 43枚/H100換算 = ~34,000 H100

インターコネクト・冷却オーバーヘッド20~30% → 保守的にH100 ~2.5~3万枚級
```

2026年現在の地球最大級AIクラスターと同等。**モジュール1基で。**

モジュールが27万基自己複製したら？H100数十億枚換算。人類全体の現在の計算能力を超える規模が小惑星一つから出る。

## 原料：スラグからAIチップが出る

ここがこの設計の白眉だ。専用の半導体鉱山は必要ない。

小惑星原鉱を精錬するとFe-Ni（90%+）が主産物で、**残りがスラグ**だ。スラグの主成分はSiO₂ — ケイ酸塩。これを捨てない。

```
小惑星原鉱 → 真空精錬
  +→ Fe-Ni (90%+) → ミラー、構造材、バッテリー、タービン
  +→ スラグ (SiO₂主成分)
       +→ 大部分 → 放射線遮蔽材
       +→ 一部 → 炭素還元 (SiO₂ + 2C → Si + 2CO)
            → 金属シリコン
            → ゾーンリファイニング（太陽熱 + 真空 + 微小重力）
            → 半導体級単結晶インゴット (9N+純度)
            → 300mm ウェハー
            → 28nm TPU
```

**精錬廃棄物からAIチップが出る。**

[ゾーンリファイニング（zone refining）](https://en.wikipedia.org/wiki/Zone_melting)が宇宙で有利な理由がある。シリコンインゴットに狭い溶融帯（molten zone）を通過させて不純物を押し出す精製法だが：

- **エネルギー：** 太陽熱直接加熱。コストゼロ
- **真空：** 宇宙が真空。不純物が自動蒸発
- **微小重力：** 溶融帯が垂れ落ちない。地球のFZ（Float Zone）法はインゴット直径200mmが限界——それ以上は溶融シリコンが重力に耐えられず崩れる。**無重力なら300mm以上も可能**
- **繰り返し：** ミラーの角度を調整するだけで精製パスを無限反復。追加コストゼロ

地球ではゾーンリファイニングは高価で小規模なプレミアム工程だ。宇宙では**基本工程**になる。

## ファブ：宇宙がクリーンルームだ

地球の28nmファブの最大コスト項目の一つ：クラス1~10クリーンルーム。空気1立方フィートあたり直径0.5μm以上のパーティクルが10個以下。これを維持するには巨大なHEPAフィルターシステム、エアハンドリングユニット、陽圧（positive pressure）管理が必要。ファブ建設費のかなりの部分がここに投じられる。

宇宙には**空気がない。** パーティクル汚染源自体が不在。真空は**完璧なクリーンルーム**だ。

7大工程段階ごとの宇宙適合性：

| 工程 | 宇宙適合性 | 理由 |
|------|-----------|------|
| インゴット成長 | **宇宙優位** | 微小重力FZ法、大口径インゴット |
| ウェハースライシング | 可能 | 機械的工程、環境無関係 |
| 酸化/蒸着 (CVD, PVD) | **真空有利** | 地球ではチャンバーを真空にする必要あり — 宇宙はすでに真空 |
| フォトリソグラフィ | ボトルネック | ArFスキャナー・フォトレジストが地球依存 |
| エッチング | **真空有利** | プラズマエッチングチャンバー簡素化 |
| イオン注入 | **真空有利** | ビーム散乱減少、高真空ポンプ不要 |
| 配線/パッケージング | 可能 | Cuは小惑星/月から調達 |

**7段階中6段階が宇宙で有利または同等。** 唯一のボトルネックはフォトリソグラフィ — ArFスキャナー自体を宇宙で作ることはできない。だが一度持ち上げれば数十年使える。

## ファブ熱管理：「宇宙で半導体を作る？」

「太陽に面した側は数百度、裏側はマイナス100度なのに±0.01°Cの制御ができるのか？」

できる。そして**地球より簡単だ。**

### 問題の核心

ArFリソグラフィスキャナーの投影レンズシステムは熱膨張に極めて敏感だ。温度0.01°Cの変動がレンズ曲率を変え、オーバーレイ（overlay）誤差を生み、歩留まりを削る。28nm工程のオーバーレイ許容誤差は数nm。

地球のファブではこれをどう解決するか：
- クリーンルーム全体を23.00 ± 0.1°Cに恒温維持
- スキャナー内部は別の冷却回路で±0.01°C制御
- **問題：** 外部からの擾乱が絶え間なく入る — 外気温変動、季節、昼夜、天候、地震、道路振動、隣接装置の発熱

### 宇宙ファブの熱設計

```
[ファブモジュール断面]

外部: 宇宙真空（伝導ゼロ、対流ゼロ）
  |
  +- MLI反射壁（多層反射断熱、数十層）
  |    → 太陽輻射熱遮断率99.5%+
  |    → 内部→外部輻射損失も遮断
  |
  +- 構造外壁 (Fe-Ni)
  |
  +- 能動液体循環層
  |    → 超純水(UPW)微循環
  |    → ポンプ + ヒーター + 放熱バルブで能動制御
  |    → 内壁全体 23.00 ± 0.05°C 均一
  |
  +- ファブ内部 (1 atm N₂ 雰囲気)
       → 装置発熱 → 循環冷媒が吸収
       → スキャナー内部: 専用冷却ループ ±0.01°C
```

### なぜ地球より簡単か

| 項目 | 地球ファブ | 宇宙ファブモジュール |
|------|--------|-------------|
| 外部温度擾乱 | 絶え間ない（天候、季節、昼夜） | **なし** — 真空断熱 |
| 外部振動 | 道路、地震、隣接工場 | **なし** — 宇宙は無振動 |
| 断熱コスト | HVACにファブ電力の30~40% | **真空がタダの断熱材** |
| 熱源予測性 | 外部擾乱 + 内部装置 | **内部装置のみ**（完全予測可能） |
| 放熱 | 冷却塔、チラー（水・電力大量消費） | **放熱パネル**（真空輻射） |

核心の逆説：宇宙の極端な熱環境（数百度 vs マイナス100度）は**ファブ内部には届かない。** 真空は最高の断熱材であり、MLIが輻射を遮断すればファブ内部は外部と熱的に完全に隔離される。そこから先は内部装置の発熱を管理するだけで、これは地球より簡単だ — 外部擾乱がゼロだから。

地球のファブがHVACに全電力の30~40%を使うのは**外部と絶えず戦っているから**だ。宇宙ファブにはその戦い自体がない。

### UPW — バソライザーから来る

ファブ恒温循環に使う超純水（UPW）は専用浄水プラントではなくバソライザー産出物から得る：

```
バソライザー: H₂O → H₂ + O₂ (電気分解)
逆反応:       H₂ + O₂ → H₂O (燃料電池)

副産物 H₂O → 精製 → UPW
  +→ ファブ恒温循環冷媒
  +→ ウェハー洗浄
  +→ イマージョンリソグラフィ液体
```

### 人工重力区画

イマージョンリソグラフィはウェハー上に超純水の薄膜を敷く必要がある — 重力が必要だ。ファブモジュールを二つの区画に分ける：

```
真空区画 (0G):
  +→ CVD/PVD蒸着（真空必須）
  +→ イオン注入（真空必須）
  +→ プラズマエッチング（真空必須）

人工重力区画 (~1G 回転):
  +→ ArFイマージョンリソグラフィ（液体管理に重力が必要）
  +→ ウエット洗浄（UPW洗浄に重力が必要）
  +→ ウェハーハンドリング（ロボット搬送）
```

ウェハーはエアロックを通じて真空区画 ↔ 人工重力区画を行き来する。回転区画は外部振動源がないため**回転自体の均一性を管理するだけでよい** — 地球で地震や道路振動まで防御するよりはるかに簡単だ。

## 外部依存：5%

| 区分 | 調達 | 備考 |
|------|------|------|
| シリコン | 現地（スラグ → Si） | |
| エネルギー | 現地（太陽熱） | |
| クリーンルーム | 現地（宇宙真空） | |
| 超純水 | 現地（バソライザー H₂O → 精製） | |
| 銅配線 | 現地（小惑星/月） | |
| ArFスキャナー | **地球1回** | 数十年の寿命 |
| フォトレジスト | **地球年1回** | 年間数百kg |
| エッチングガス | **地球年1回** | 再利用、少量 |
| ドーピング元素 (B, As) | **地球年1回** | 数十kg |

95%は宇宙で調達。残り5% — ArFスキャナー（初回1回）+ 消耗品（年間数トン）— はスターシップ1回の打ち上げで数十年分を搭載可能。

「フォトレジストは精密有機化学では？」——その通り。これは現地製造が難しい。しかし年間消費量は数百kgレベルだ。スターシップ1回で数十年分を運べる。完全自給ではなく**事実上の自給**だ。

## 自己複製ループが閉じる

```
これまで:
  小惑星原鉱 → 精錬 → Fe-Ni → ミラー・構造材・バッテリー → 自己複製
                                                          ↑
                                                  AIチップは地球から輸入

これから:
  小惑星原鉱 → 精錬 → Fe-Ni → ミラー・構造材・バッテリー・タービン
                    → スラグ → Siインゴット → 28nm TPU → AI自律制御
                                                          ↓
                                                  自己複製ループ完全閉鎖
```

ミラーがミラーを作る。バッテリーが推進剤を作る。**スラグがAIチップを作る。** 捨てるものがない。

## 一行まとめ

最先端3nmはASML独占のEUVなしには作れない — 宇宙では不可能。28nmはArFだけで可能で、Google TPU v1が実測92 TOPSを証明した。電力4.6倍の劣位は370 MWモジュールではミラー2.4 m²の差。スラグからシリコンが出て、宇宙そのものがクリーンルームであり、真空断熱が±0.01°C熱管理を地球より簡単にする。自己複製ループの最後の環。
