## 前回の宿題

前回の記事で、タービンはPVより自己複製に有利だと述べた。効率30%、発電370 MW、残り855 MWは熱。

そしてこう書いた：

> 「同じ70%が製錬所→工場→居住区→データセンターを順次通過し、すべて使われる。」

概念的には正しい。タービン廃熱はPVの60°C低温廃熱よりはるかに有用だという点は変わらない。**しかし「順次通過」は実際の設計ではない。** 今回は本当の熱の流れを追う。

---

## まず訂正：「順次通過」はなぜ成り立たないか

### 問題1：タービン廃熱の温度

タービン（Braytonサイクル）の熱力学：
- 高温側：~1,200°C（ミラー集光で作動流体を加熱）
- 低温側：~227°C（ここで排熱）
- 効率30% → 370 MW電力、**855 MWが~227°Cで排出**

要点：**タービン廃熱はすべて~227°C。** 製錬には1,600°Cが必要。227°Cの熱で1,600°Cのプロセスは動かせない——熱力学第二法則。熱は高温から低温にしか流れない。

前回の図で「800~1,000°C→製錬」と書いた部分はタービン廃熱ではない。製錬の熱は**ミラーから直接**供給される。

### 問題2：1,000°Cを運べる媒体がない

仮に1,600°Cの熱がどこかにあっても、パイプで別の施設に送れるか？

| 熱媒体 | 最高運用温度 | 制限 |
|---|---|---|
| 加圧水 | ~340°C | 臨界点 |
| 溶融塩 | ~565°C | 分解 |
| 液体ナトリウム | ~800°C | 蒸気圧 |
| 高圧ヘリウム | ~950°C | 配管材料限界 |
| **1,000°C以上** | **該当なし** | **媒体なし** |

1,600°Cの熱を運べる流体は存在しない。この温度でエネルギーを伝える方法はただ一つ：**光。** ミラーによる直接照射。

### 問題3：モジュール間の距離

特化クラスタにおいて、製錬モジュールとデータセンターモジュールは**50~100 km離れている。** 振動・汚染・熱干渉防止のための意図的な離隔だ。この距離で熱配管は非現実的。

**結論：タービン廃熱を高温プロセスに送ることは物理的に不可能。**

---

## 実際の設計：各施設が専用ミラーを受ける

熱の流れの本当の原則：

1. **入力熱は各モジュールが自分のミラーから直接受ける** ——光で伝達、媒体不要
2. **カスケードは各モジュール内部でのみ機能** ——プロセス廃熱を段階的に低温で再利用
3. **モジュール間で熱は伝達しない** ——距離と媒体の制約
4. **100°C未満の低温廃熱のみ居住区に供給** ——配管可能、居住区の需要と温度が一致

### ミラー配分（10基クラスタ基準）

| モジュール種別 | 数量 | ミラー配分（熱：電力） | 高温熱源 |
|---|---|---|---|
| 製錬モジュール | 3基 | 90:10 | ミラー → 直接1,600°C |
| インゴットモジュール | 1基 | 70:30 | ミラー → 直接1,400°C |
| 構造材モジュール | 2基 | 60:40 | ミラー → 直接800~1,200°C |
| ファブモジュール | 1基 | 20:80 | ミラー → 直接900°C |
| データセンター | 2基 | 5:95 | ミラー → タービン → 電力 |
| 居住/物流 | 1基 | 30:70 | ミラー → タービン → 電力 |

**1,000°C以上は光が直接届ける。** タービンは電力が主に必要なモジュール（データセンター、居住区）でのみ稼働。

## 放熱板の物理：T⁴法則

宇宙で熱を捨てる方法はただ一つ：**赤外線放射。** 対流も伝導もない。

Stefan-Boltzmann法則：

**放射電力 = ε × σ × A × T⁴**

（ε：放射率、σ：Stefan-Boltzmann定数、A：面積、T：絶対温度）

鍵は**T⁴**。温度が2倍なら放射量は16倍。逆に、同じ熱量を捨てるのに必要な面積は1/16。

| 放熱温度 | 1 MWあたり必要面積 | 例え |
|---|---|---|
| 800°C (1,073K) | **8 m²** | 駐車場1台分 |
| 400°C (673K) | **50 m²** | マンション1戸 |
| 227°C (500K) | **166 m²** | テニスコート |
| 100°C (373K) | **535 m²** | バスケットコート3面 |
| 60°C (333K) | **844 m²** | サッカー場1/8 |

（両面放射、放射率ε = 0.85、無塗装Fe-Ni板材基準）

**教訓：800°Cで捨てれば8 m²で済む熱が、60°Cまで下がると844 m²必要になる。100倍以上。**

したがって熱管理の核心原則：**「使えない熱は、可能な限り高い温度で即座に捨てる。」**

### 放熱板の素材

放熱板も自己複製ループの中にある：
- **素材：** 小惑星産Fe-Ni薄板
- **表面：** アルミコーティングなし（ミラーと逆）——無塗装Fe-Niは赤外線放射率が高く、放熱に最適
- **製造：** ミラーフレームと同じ板金ラインで生産。コーティング工程を省略するだけ
- **追加リソース：** ゼロ。同じ素材、同じ工程、用途だけが違う

## 施設別の熱の流れ

### 製錬モジュール——熱が主役（熱90%、電力10%）

製錬モジュールはミラーエネルギーの90%を熱として直接受ける。小型タービン（10%）がモーター・ロボット用の電力を生産。

```
☀️ 専用ミラー（90% → 直接照射、10% → 小型タービン）
 │
 ▼
製錬炉 (1,600°C) ← ミラー光で直接加熱、媒体不要
 │
 │ 廃熱 ~800°C ← ここから媒体（He/液体金属）で伝達可能
 ├→ 合金熱処理、アニーリング（800°C活用）
 ├→ 余剰 → ★ 放熱板A (800°C) — 8 m²/MW、コンパクト
 │
 │ 廃熱 ~400°C
 ├→ 予熱、補助加熱（400°C活用）
 ├→ 余剰 → ★ 放熱板B (400°C) — 50 m²/MW、中型
 │
 │ 廃熱 ~200°C
 ├→ ★ 放熱板C (200°C) — 大部分をここで処分
 │
 │ 残余 <100°C
 └→ 居住区へ供給可能（配管で伝達）

小型タービン廃熱（~227°C）→ ★ 放熱板D
```

製錬モジュールは高温から段階的に熱を使い、**各段階で余剰を即座に放熱板から放出。** 高温放熱板は面積が小さいため負担は軽い。最後に100°C以下になった残余熱だけを居住区に送る。

### データセンターモジュール——電力が主役（熱5%、電力95%）

データセンターは放熱が最も難しいモジュール。ミラーエネルギーの95%がタービン→電力→チップ→熱に変換され、その熱がすべて~60°Cで出てくる。

```
☀️ 専用ミラー（95% → 大型タービン、5% → 補助熱）
 │
 ▼
大型タービン → ~370 MW級発電
 │
 │ タービン廃熱 ~227°C（~855 MW）
 └→ ★ 放熱板A (227°C) — 166 m²/MW
     タービン廃熱の大部分をここで処分

チップ稼働 → 電力がすべて熱に変換
 │
 │ チップ発熱 ~60°C
 │  60°C直接放射時：844 m²/MW → 111 MWで~94,000 m²必要
 │
 ├→ [ヒートポンプ] 60°C → 200°Cに昇温（COP ~3、電力~37 MW）
 │   └→ ★ 放熱板B (200°C) — 面積~1/4に縮小
 │
 └→ 残余 <100°C → 居住区へ供給可能
```

**ヒートポンプが鍵。** 60°Cの熱を200°Cに上げて放射すれば、放熱板面積が大幅に縮小する。ヒートポンプの電力（~37 MW）はタービン出力から賄う。タービンもヒートポンプもFe-Ni + Tiで現地製造可能。

### 構造材モジュール（熱60%、電力40%）

```
☀️ 専用ミラー（60% → 直接加熱、40% → タービン）
 │
 ▼
溶接・熱処理 (800~1,200°C) ← ミラー直接加熱
 │ 廃熱 ~400°C
 ├→ 成形・ベンディング予熱（400°C活用）
 ├→ 余剰 → ★ 放熱板 (400°C)
 │ 廃熱 ~200°C
 ├→ ★ 放熱板 (200°C)
 │ 残余 <100°C
 └→ 居住区へ供給可能

タービン (40%) → 電力（ロボット、CNC、溶接機）
 └→ タービン廃熱 → ★ 放熱板 (227°C)
```

### 居住/物流モジュール——100°C未満廃熱の消費者

居住モジュールは熱の最終消費者だ。自前のタービンで生命維持・照明・農業用電力を生産しながら、**近隣モジュールから100°C未満の低温廃熱を受け取る。**

```
☀️ 専用ミラー（30% → 熱、70% → タービン）
 │
 ├→ タービン → 電力（生命維持、照明、農業LED）
 │   廃熱（~227°C）→ ★ 放熱板
 │
 └→ 熱 → 温水、暖房補助
     └→ 残余 → ★ 放熱板

近隣モジュール（製錬、構造材）の<100°C廃熱
 │
 └→ 居住区暖房、温水、農業土壌加温
     └→ 残余 → 居住区外壁放熱（構造物自体が放熱板の役割）
```

居住区の熱需要（暖房、温水）は、産業モジュールの廃熱規模に比べてささやかだ。近隣モジュールの100°C未満残余熱だけで十分に賄える。**居住区が無料暖房を受けているのであって、産業モジュールが居住区のために熱を作っているのではない。**

## 分散放熱：全体像

クラスタ全体の熱の流れをまとめると：

```
☀️ 太陽光 → ミラー → 各モジュールに直接配分
                    │
    ┌───────────────┼───────────────┐
    ▼               ▼               ▼
[製錬モジュール] [構造材モジュール] [データセンター]
 ミラー→1,600°C  ミラー→1,200°C   ミラー→タービン→電力
    │               │               │
    ▼               ▼               ▼
 ★放熱(800°C)   ★放熱(400°C)   ★放熱(227°C) ← タービン廃熱
 ★放熱(400°C)   ★放熱(200°C)   ★放熱(200°C) ← ヒートポンプ後
 ★放熱(200°C)       │               │
    │               ▼               ▼
    └──── <100°C ──→ [居住モジュール] ←── <100°C
                      暖房・温水
                         │
                    ★放熱(外壁, ~30°C)
```

**「順次通過」ではなく「並列配分＋個別放熱＋低温のみ共有」。** 各モジュールが自分のミラーから熱を受け、自分の放熱板で熱を捨て、残りかすだけを居住区に渡す。

### なぜこの方が優れているか

1. **高温放熱板は小さい** ——800°Cで1 MWを捨てるのに8 m²。高温プロセスの横に小さなフィンを付けるだけ
2. **モジュール間配管がない** ——50 km高温配管の悪夢を回避
3. **各モジュールが熱的に独立** ——1モジュールのメンテナンスが他に影響しない
4. **居住区は安全** ——1,600°Cの熱配管が居住区を通らない

## 前回の修正：タービンの70%はどこへ行くのか

前回「PVは70%を捨て、タービンは使う」と書いた。これはまだ正しいか？

**正しい。** ただしメカニズムが異なる：

| | PV | タービンシステム |
|---|---|---|
| 30% | 電力 | 電力 |
| 残り70% | 60~80°C低温廃熱 → **使い道なし** | ミラー直接加熱で各プロセスに配分 → **製錬・成形・熱処理に直接使用** |
| 放熱負担 | 70%すべてを低温で放熱（巨大な放熱板） | 高温で段階的に放熱（小型分散放熱板） |

PVの70%はすべて60~80°C——産業にも放熱にも最悪の温度。タービンシステムでは、70%がミラーを通じて各プロセスに必要な温度で直接投入され、廃熱は可能な限り高温で放射される。

**「残り70%を使う」の本当の意味：タービン廃熱ではなく、ミラーの熱エネルギーを各プロセスが直接消費する。**

---

## 一行まとめ

1,600°Cをパイプで運べる媒体はない。だから各施設がミラーを直接受ける。熱は各プロセス内で段階的に使われ、余剰は可能な限り高い温度で放熱板から即座に捨てる。居住区には100°C未満の残余廃熱のみ供給。放熱板はミラーフレームと同じFe-Ni板材——コーティングを省くだけで放熱板になる。
