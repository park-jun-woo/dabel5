## 1. 전략 요약

DABEL 데이터센터의 AI 연산을 자급하기 위한 반도체 전략.
최첨단 공정(3~5nm)을 추격하지 않고, **성숙 공정(28nm)으로 단순한 AI 가속기를 대량 생산**하여 물량으로 최첨단 성능을 달성한다.

| 원칙 | 내용 |
|------|------|
| 공정 | **28nm** (ArF 이머전, EUV 불필요) |
| 아키텍처 | **시스톨릭 어레이** (Google TPU v1 스타일, AI 행렬 연산 특화) |
| 클럭 | **700 MHz~1 GHz** (안정성 우선) |
| 성능 보상 | **물량 투입** (전력 제약 없음) |
| 핵심 이점 | 태양열 무한 전력, 진공 클린룸, 미소중력 잉곳 성장 |

## 2. 왜 28nm인가

### 2.1. 28nm의 전략적 위치

| 항목 | 28nm | 3~5nm (최첨단) |
|------|------|---------------|
| 리소그래피 | **ArF 이머전** (니콘, 캐논, ASML 모두 생산) | EUV (ASML 독점, 대당 수천억 원) |
| 장비 가용성 | **성숙 시장, 중고 장비 풍부** | 극히 제한적, 수출 규제 대상 |
| 설계 복잡도 | 낮음 (싱글 패터닝) | 극히 높음 (멀티 패터닝) |
| 팹 건설 비용 | ~$3~5B | ~$20~30B |
| 특허/IP | 성숙, 접근 용이 | 첨예한 특허 경쟁 |
| 수율 | 높음 (검증된 공정) | 초기 수율 저조 |

**28nm는 "누구나 만들 수 있는 가장 강력한 공정"이다.** EUV 없이 도달 가능한 최고 밀도이며, 중국을 포함한 다수 국가가 28nm 팹을 보유하고 있다.

### 2.2. 레퍼런스: Google TPU v1 (실측 데이터)

Google이 2015년에 28nm로 실제 생산하여 10만 개 이상 배포한 AI 가속기.

| 항목 | Google TPU v1 (실측) |
|------|---------------------|
| 공정 | 28nm |
| 다이 면적 | ≤ 331 mm² |
| 구조 | 256 × 256 시스톨릭 어레이 |
| 클럭 | 700 MHz |
| 연산 | 92 TOPS (INT8) ≈ 23 TFLOPS (FP16 환산) |
| 소비전력 | 28~40W (TDP), 실운용 ~75W |
| 온칩 메모리 | 28 MiB SRAM + 4 MiB 누산기 |
| 외장 메모리 | 8 GiB DDR3 (34 GB/s) |
| 실리콘 활용률 | **90%+ (GPU는 ~30%)** |

시스톨릭 어레이는 실리콘의 90% 이상이 실제 연산을 수행한다. GPU는 범용성을 위한 제어 로직, 캐시, 스케줄러 등에 70%를 소비한다. AI만 돌릴 거면 이 오버헤드가 전부 낭비.

## 3. H100급 연산 달성 계산

### 3.1. 기준점: NVIDIA H100

| 항목 | NVIDIA H100 SXM |
|------|-----------------|
| 공정 | 4nm (TSMC) |
| FP16 연산 | ~990 TFLOPS |
| 소비전력 | 700W |
| HBM3 | 80 GB, 3.35 TB/s |
| 가격 | ~$25,000~40,000 |

### 3.2. TPU v1 스펙 기준 H100 대체

990 TFLOPS ÷ 23 TFLOPS = **43장**

43 × 75W = **3,225W ≈ 3.2 kW**

| 항목 | DABEL (TPU v1 × 43장) | NVIDIA H100 × 1장 |
|------|----------------------|-------------------|
| FP16 합산 | ~990 TFLOPS | ~990 TFLOPS |
| 총 소비전력 | **3.2 kW** | 700W |
| 전력 비율 | **4.6배** | 기준 |
| 다이 총면적 | ~14,233 mm² | 814 mm² |

**전력 4.6배 열위는 DABEL에서 무의미.** 370 MW 중 3.2 kW는 반올림 오차.
거울 면적 2.4 m²만 더 펴면 되는 전력.

### 3.3. 메모리 문제

H100의 진짜 무기는 80 GB HBM3 (3.35 TB/s). 28nm로는 HBM 제조 불가.

**대안:**

| 방식 | 용량 | 대역폭 | 적합 워크로드 |
|------|------|--------|-------------|
| 온칩 SRAM 탑재 | 칩당 수백 MB | 극히 높음 (온칩) | 추론 (inference) |
| 별도 DRAM 모듈 | 칩당 수 GB | 중간 (인터포저) | 소규모 학습 |
| 멀티칩 SRAM 풀 | 43장 합산 수십 GB | 칩 간 연결 의존 | 대규모 추론 |

28nm 칩 43장의 합산 메모리가 H100의 80 GB에 못 미치지만, **AI 추론(inference)에는 충분**. 대규모 학습(training)은 메모리 제약이 있으나, DABEL의 주 용도인 시스템 최적화·과학 연산·추론 서비스에는 적합.

### 3.4. DABEL 모듈 1기 데이터센터 규모

전력 배정: 111 MW (모듈 전체 370 MW의 30%)

| 항목 | 값 |
|------|-----|
| TPU v1급 칩 총 수량 | **~1,480,000장 (148만 장)** |
| H100 환산 | **~34,000장급** |
| FP16 합산 | **~34 EFLOPS** |
| 오버헤드 20~30% 감안 | **H100 ~24,000~27,000장급** |

현재 지구 최대급 AI 클러스터와 동급.
20만 기 자기복제 시 **H100 수십억 장급** 도달.

## 4. 우주 실리콘 잉곳 생산: 존 리파이닝

### 4.1. 원료 확보

```
소행성 원광 (1986 DA)
    ↓ 진공 제련 (기존 DABEL 공정)
    ├→ 철-니켈 (구조재, 거울, 배터리)
    ├→ 백금족 (파이프 코팅, 거울 보호막, 배터리 촉매)
    ├→ 슬래그 대부분 → 차폐재 (기존 용도 유지)
    └→ 슬래그 일부 (SiO₂ 주성분)
         ↓ 탄소 환원 (태양열, SiO₂ + 2C → Si + 2CO)
         금속 실리콘 (MG-Si, 순도 ~98%)
         ↓ 존 리파이닝 / 플로트 존 (태양열 + 진공 + 미소중력)
         반도체급 단결정 실리콘 잉곳 (순도 99.9999999%, 9N+)
         ↓ 웨이퍼 슬라이싱
         실리콘 웨이퍼 (300mm)
```

**제련 폐기물(슬래그)에서 반도체가 나온다.** DABEL 자원 철학의 극치.

### 4.2. 존 리파이닝이 우주에서 유리한 이유

| 요소 | 지구 | DABEL (우주) |
|------|------|------------|
| **에너지** | 전기로 (비용 높음) | **태양열 직접 가열 (비용 제로)** |
| **진공** | 진공 챔버 구축 필요 | **우주가 진공 (공짜)** |
| **중력** | 용융대 흘러내림 → 대구경 한계 (~200mm) | **미소중력 → 용융대 안정 → 300mm+ 가능** |
| **불순물 탈기** | 챔버 내 진공도 한계 | **우주 진공에서 휘발성 불순물 자동 증발** |
| **반복 패스** | 패스당 에너지 비용 누적 | **거울 각도만 조절 → 무한 반복 비용 제로** |
| **도가니 오염** | FZ법으로 회피 가능하나 중력 제약 | **FZ법 + 미소중력 = 도가니 없이 대구경 초고순도** |

**플로트 존(FZ) 법:**
- 도가니 없이 잉곳을 세로로 세우고 용융대를 이동
- 도가니 오염 제로 → 초고순도 달성
- 지구에서는 중력 때문에 200mm가 한계
- **미소중력에서는 300mm 이상 가능** → 표준 웨이퍼 크기 직접 생산

### 4.3. 탄소 조달

SiO₂ → Si 환원에 탄소 필요.

| 공급원 | 가용성 | 비고 |
|--------|--------|------|
| C형(탄소질) 소행성 | 소행성대에 풍부 | 장기적 최적 공급원 |
| 지구 조달 | 스타십으로 수송 | 초기 단계 |
| 제련 부산물 | 소행성 원광 내 탄소 미량 | 보조적 |

## 5. 28nm 팹 우주 구축: 공정별 가능성 평가

### 5.1. 7대 공정 단계 분석

| 공정 단계 | 우주 적합성 | 핵심 이점 / 제약 |
|----------|-----------|----------------|
| **① 잉곳 성장** | ⭐ **우주 우위** | 미소중력 FZ법, 진공 탈기, 태양열 |
| **② 웨이퍼 슬라이싱** | ✅ 가능 | 기계적 공정, 난이도 낮음 |
| **③ 산화/증착** | ⭐ **진공 유리** | CVD/PVD 진공 챔버 불필요, 진공 펌프 불필요 |
| **④ 포토리소그래피** | ⚠️ **최대 병목** | ArF 스캐너·레티클·포토레지스트 지구 의존 |
| **⑤ 에칭** | ⭐ **진공 유리** | 플라즈마 에칭 챔버 단순화 |
| **⑥ 이온 주입** | ⭐ **진공 유리** | 빔 경로 산란 감소 → 정밀도 향상 |
| **⑦ 배선/패키징** | ✅ 가능 | Cu는 우주 조달 가능 |

### 5.2. 클린룸: 우주 자체가 클린룸

지구 28nm 팹에서 가장 비싼 인프라 중 하나가 클래스 1~10 클린룸.

- 지구: 공기 중 파티클을 극도로 제거해야 함 → 막대한 HVAC 비용
- **우주: 애초에 공기가 없음** → 밀폐 챔버를 진공 유지만 하면 파티클 오염원 자체 부재
- 주의: 장비 아웃개싱, 기계 마모 파티클은 별도 관리 필요

### 5.3. 포토리소그래피 상세 (최대 병목)

28nm는 **ArF(193nm) 이머전 리소그래피** 사용. EUV 불필요.

**우주에서 유리한 점:**
- 진동 환경: 외부 진동원(도로, 지진) 없음 → 리소그래피 정밀도 향상
- 온도 안정성: 진공 단열로 온도 변동 최소화 가능

**우주에서 어려운 점:**
- ArF 엑시머 레이저 광원: 우주 제조 극히 어려움 → **지구 조달**
- 초정밀 렌즈 시스템: 수십 년 광학 노하우 집약 → **지구 조달**
- 이머전 액체(초순수): 미소중력 관리 필요 → **오닐 실린더 인공중력 내 배치로 해결**
- 레티클(마스크): 칩 설계 변경 시에만 필요 → **지구 조달 (빈도 극히 낮음)**
- 포토레지스트: 유기 고분자, 현지 합성 곤란 → **지구 정기 보급 (소량)**

**핵심 판단:** ArF 스캐너는 한 번 올리면 수십 년 사용. 소모품(레지스트, 가스)은 연간 수 톤 이하. 장비와 소모품 합쳐서 총 외부 의존도 ~5%.

### 5.4. 화학물질 조달 매트릭스

| 물질 | 용도 | 우주 조달 | 비고 |
|------|------|----------|------|
| 초순수 (UPW) | 세정, 이머전 | ✅ | 바톨라이저 H₂+O₂ → H₂O → 정제 |
| SiH₄ (모노실란) | CVD 증착 | ⚠️ | Si + H₂에서 합성 가능 |

| N₂ | 분위기 가스 | ⚠️ | 소행성 가스 또는 지구 조달 |
| HF (불산) | 세정, 에칭 | ⚠️ | 불소(F) 조달 과제 |
| H₂SO₄ | 세정 | ⚠️ | 황은 제련 부산물에서 포집 |

| 도핑 B (붕소) | P형 도핑 | ❌ | **지구 조달 (극소량)** |
| 도핑 As (비소) | N형 도핑 | ❌ | **지구 조달 (극소량)** |

## 7. 반도체 자급 로드맵: 4단계

### Genesis Module과 전문화 클러스터

DABEL의 모듈 아키텍처는 **Genesis Module(만능 1기) → 전문화 클러스터(10기)**로 분화한다. 반도체 팹은 전문화 클러스터의 **팹 모듈(1기)**이 전담한다.

```
[지구에서 발사]
     │
     ▼
  Genesis Module (만능 1기, 줄기세포)
  ┌─────────────────────────────────┐
  │ 소형 제련 + 구조재 + 발전 + 거주  │
  │ + 기본 AI 서버 (지구산 칩)       │
  │ → 1~3년간 다음 10기 부품 생산    │
  └──────────────┬──────────────────┘
                 │
                 ▼
  1세대 전문화 클러스터 (10기, 분화세포)
  ┌──────────────────────────────────────┐
  │ 제련 ×3 → 잉곳 ×1 → 팹 ×1          │
  │ 구조재 ×2   데이터센터 ×2   허브 ×1   │
  └──────────────────────────────────────┘
```

**팹 모듈의 환경 최적화:**
- 거울 배분 20% 열 / 80% 전력 (리소·에칭·이온주입은 전기)
- 산화/CVD 가열은 거울 직접 조사 (900~1,200°C)
- 극저진동 설계 (제련 모듈과 50km+ 이격)
- 우주 진공 = 클린룸 (파티클 오염원 부재)
- 오닐 실린더 내부에 인공중력 → 이머전 리소그래피 가능

**모듈 간 물류:**
- 잉곳 모듈 → 팹 모듈: Si 웨이퍼 배달 (50km, 수 m/s, 에너지 ~500 J/톤)
- 팹 모듈 → 데이터센터 모듈: TPU 칩 배달
- 델타-V ~0.1~1 m/s → 물류 비용 거의 제로

### Phase 1: 지구 조달 (현재~2040년대)

칩을 전량 지구에서 제조하여 우주로 수송.
28nm TPU급 칩을 지구 파운드리(삼성, TSMC, GlobalFoundries)에서 양산.
스타십으로 대량 수송.

**이 단계에서도 DABEL 데이터센터는 가동 가능.** 칩만 올리면 전력과 냉각은 현지.

### Phase 2: 잉곳·웨이퍼 현지 생산 (2040~2060년대)

슬래그에서 실리콘 추출 → 존 리파이닝 → 웨이퍼 생산을 우주에서 수행.
칩 제조(팹 공정)는 지구에서 계속.
**웨이퍼를 우주에서 만들어 지구로 보내고, 팹 공정 후 칩을 다시 올리는** 하이브리드 모델도 가능.
(우주산 초고순도 웨이퍼가 지구산보다 품질이 높을 수 있음)

### Phase 3: 반자급 팹 (2060~2080년대)

ArF 스캐너 등 핵심 장비를 우주에 설치하고, 팹 공정 95%를 우주에서 수행.
포토레지스트, 에칭 가스, 도핑 원소 등 **5% 소모품만 지구에서 연 1회 보급.**
같은 칩 설계(TPU 아키텍처)를 대량 생산 → 레티클 교체 빈도 극히 낮음.

**"28nm 팹의 95%는 우주에서, 5%는 스타십이 연 1회 배달."**

### Phase 4: 완전 자급 (2080~2100년대+)

소행성대 대규모 채굴 활성화로 모든 원소 조달 가능.
포토레지스트 현지 합성 (탄소 화학 산업 확립).
에칭 가스 현지 합성.
도핑 원소 소행성에서 채굴.
ArF 레이저/렌즈 현지 제조 (광학 산업 확립).

**100년 내 100% 자급 달성 가능.** 외부 의존 제로.

## 9. 기존 DABEL 시스템과의 연계

### 모듈 아키텍처 연계

| DABEL 설계 | 반도체 팹과의 관계 |
|-----------|-----------------|
| **Genesis Module** | 지구산 칩으로 초기 AI 서버 운용, 1세대 팹 모듈 부품 생산 |
| **제련 모듈 (×3)** | 슬래그(SiO₂) → 잉곳 모듈로 배달 (반도체 원료) |
| **잉곳 모듈 (×1)** | SiO₂ → 탄소환원 → 존 리파이닝 → Si 웨이퍼 → 팹 모듈로 배달 |
| **팹 모듈 (×1)** | Si 웨이퍼 → 28nm TPU 생산 → 데이터센터 모듈로 배달 |
| **데이터센터 모듈 (×2)** | TPU 148만 장 탑재, H100 3만장급 AI 연산 |
| **구조재 모듈 (×2)** | 팹 장비 프레임, 진공 챔버 구조물 제작 |
| **거주/물류 허브 (×1)** | 모듈 간 화물 중계, 팹 정비 인력 거점 |

### 기존 인프라 활용

| DABEL 기존 설계 | 팹 모듈에서의 활용 |
|---------------|-----------------|
| 태양열 (거울 직접 조사) | 산화 공정 (1,200°C), CVD 가열 (900°C) |
| 터빈 발전 | 리소그래피, 에칭, 이온주입 등 전기 장비 |
| 진공 환경 | 클린룸 대체, CVD/PVD/에칭 챔버 단순화 |
| 미소중력 | FZ 대구경 잉곳 (잉곳 모듈) |
| 오닐 실린더 (인공중력) | 이머전 리소그래피 운용 (팹 모듈) |
| 바톨라이저 | 초순수(H₂O), 산소(O₂), 수소(H₂) 공급 |
| 방열 시스템 | 팹 장비 냉각 (각 모듈 독립 방열판) |
| 스타십 보급 | 연 1회 소모품 배달 (수 톤 이하) |
| **모듈 간 물류** | **잉곳→팹→데이터센터 배달 (델타-V ~1 m/s, 비용 ~0)** |

**반도체 팹이 기존 DABEL 클러스터 위에 자연스럽게 올라간다.** 새로운 대규모 인프라가 아니라, 전문화 클러스터의 한 모듈로서 기존 물류·에너지·자원 네트워크에 통합.

## 11. 미해결 과제

**잉곳/웨이퍼 관련:**
- 탄소 조달 경로 확정 (C형 소행성 vs 지구 수송)
- FZ 잉곳 최대 달성 가능 직경 (미소중력 유체 시뮬레이션)
- 존 리파이닝 반복 횟수 vs 도달 순도 정량 모델

**팹 공정 관련:**
- ArF 스캐너 우주 운용 시 진동/열 관리 상세 설계
- 이머전 리소그래피의 인공중력 환경 적합성 검증
- 포토레지스트 소모량 정밀 추정 (연간 보급 계획)
- 에칭 가스 재활용률 정량 분석

**화학물질 관련:**
- 모노실란(SiH₄) 우주 합성 경로 및 안전성
- 불산(HF) 조달 전략 (불소 광물 탐색)
- 포토레지스트 현지 합성 가능 시점 추정

**아키텍처 관련:**
- DABEL-TPU 상세 설계 (TPU v1 기반 최적화 방향)
- 메모리 병목 해소 전략 (SRAM 규모 vs 외장 DRAM)
- 칩 간 인터커넥트 설계 (시스톨릭 어레이 클러스터링)

**경제성 관련:**
- 우주 팹 vs 지구 팹+수송의 손익분기점 (생산량 기준)
- Phase별 총비용 추정 (장비 수송비 + 소모품 보급비)

## 12. 결론

DABEL의 반도체 전략은 "최첨단을 쫓는 것"이 아니라 **"검증된 기술을 우주의 물리적 이점과 결합하여 물량으로 압도하는 것"**이다.

28nm는 구형이지만, 우주에서는 이것이 정답이다:
- 장비가 접근 가능하고 (EUV 불필요)
- 설계가 단순하고 (수율 높음)
- 전력 열위를 태양이 보상하고 (공짜)
- 진공이 공정을 단순화하고 (클린룸 불필요)
- 미소중력이 잉곳 품질을 올리고 (FZ법)
- 원료가 제련 폐기물에서 나온다 (슬래그 → 실리콘)

**슬래그에서 시작해서 AI 칩으로 끝나는 자원 순환.**
이것이 DABEL의 마지막 외부 의존을 닫는 열쇠다.

> **"구형이지만 확실한 칩을, 전력 걱정 없이, 물량으로."**
> **"28nm 팹의 95%는 우주에서, 5%는 스타십이 연 1회 배달."**
> **"100년 내 100% 자급."**
